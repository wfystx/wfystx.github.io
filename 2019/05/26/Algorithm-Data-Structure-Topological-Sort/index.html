Algorithm & Data Structure -- Topological Sort

Posted on 2019-05-26 | Edited on 2019-07-26 | In Algorithm & Data Structure | Comments:

  • Defination of Topological Sort:

    The defination of Topological Sort in Wiki is:

    “A topological sort or topological ordering of a directed graph is a linear ordering of its vertices such that for every directed edge uv from vertex u to vertex v, u comes before v in the ordering.”

    Topological sort is used to solve dependency resolution of directed graph.

    For example, if we put a bunch of tasks in a directed graph, directed edges represent that some tasks have to be done before other tasks. With the help of topological sort, we can get the order. Another possibility is that this kind of topological sort doesn’t exist at all, in this case, we can’t get a correct order to finish these tasks. This is what we called circular dependency.

  • Prerequisite of Topological Sort:

    We can get the topological sort of a graph if and only if the directed graph is a directed acyclic graph (DAG). Never the less, every DAG has at least one way of topological sort.

  • Implementation of Topological Sort:

    Topological sort has a linear time complexity O(n), which means, we can find a topological sort in O(n) time or we can know that there is at least one cycle in the graph.

    Here is an example of directed acyclic graph:

    According to the directions in the graph above, we noticed that if we want to topological sort it, we need conditions below:

    1. Vertex 1 is before Vertex 2 and 3.
    2. Vertex 2 is before Vertex 3 and 4.
    3. Vertex 3 is before Vertex 4 and 5.
    4. Vertex 4 is before Vertex 5.

    So one of the topological sort is [1,2,3,4,5].

    If we remove the edge between vertex 4 and 5:

    We can get two different ways of topological sort: [1,2,3,4,5] and [1,2,3,5,4].

    The first vertex in topological sorting is always a vertex with in-degree as 0 (a vertex with no incoming edges). Because only these vertices without in-degree edge could be the start of topological sort.

    The problem could be solved by depth-first-search or breadth-first-search. Take BFS as example, we need to store the number of in-degree of every vertex and pick the 0 in-degree in every level of search:

    1. Create a vector inDegree to store the number of in-degree.
    2. Get all edges and add them to inDegree.
    3. Pike a vertex with 0 in-degree and start search.
    4. Subtract every child vertex of the picked vertex by 1.
    5. Repeat step 3 until all vertices are searched.
    6. If we can’t search through all vertices, then there is no topological sort way, the graph is not a DAG.
