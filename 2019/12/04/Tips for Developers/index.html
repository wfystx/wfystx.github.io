Git
Some convenience helpers in .gitconfig:
[alias]

# To display a short and brief commit history, one commit per line
hist = log --pretty=format:'%C(yellow)[%ad]%C(reset) %C(green)[%h]%C(reset) | %C(red)%s %C(bold red){{%an}}%C(reset) %C(blue)%d%C(reset)' --graph --date=short
# To undo `git add`
unstage = reset HEAD --
# `git lint` or `git lint PATH` to invoke cpplint
lint = !bash -c '.git/hooks/pre-commit $@' -

Bash
Some convenience helpers in .bashrc:

Erase duplicated item(s) with the current cmdline from the history

HISTCONTROL=erasedups

Ignore case in auto-completion

bind ‘set completion-ignore-case on’
alias – -=’cd -‘
alias ..=’cd ..’
alias …=’cd ../..’
alias ….=’cd ../../..’

mkdir and cd

function mkcd {
if [[ $# -eq 1 ]]; then
mkdir -p $1 && cd $1
else
echo “mkcd

“ && return 1
fi
}
编译相关
分离式编译
$ export source_dir=/path/to/nebula/source/directory
$ export build_dir=/path/to/buiding/directory
$ mkdir -p $build_dir/{debug,release,asan,clang}
$ cd $build_dir/debug
$ cmake $source_dir
$ make -jN && ctest -jN
$ cd $build_dir/release
$ cmake -DCMAKE_BUILD_TYPE=Release \
-DCMAKE_INSTALL_PREFIX=/usr/local \
-DENABLE_TESTING=OFF $source_dir
$ make -jN && make install
$ make DESTDIR=$HOME/nebula_install install
$ cd $build_dir/asan
$ cmake -DENABLE_ASAN=ON $source_dir
$ make -jN && make VERBOSE=1
$ cd $build_dir/clang
$ CXX=clang++ CC=clang cmake $source_dir
$ make -jN && make -jN nebula-metad
测试相关
CTest

Print what tests print out

$ ctest –verbose

Run tests concurrently

$ ctest -j8

Print verbose outputs only if testing failed

$ ctest –output-on-failure # Or, CTEST_OUTPUT_ON_FAILURE=1 ctest …

Run tests in random order

$ ctest –schedule-random

Run specific tests matching some regex

$ ctest -R status # Run status_test, status_or_test, etc.

Run test(s) by labels matching some regex

Currently we label tests by their hosting directories,

such as graph, dataman, common, common/base, etc.

$ ctest -L dataman # run tests in src/dataman/test
$ ctest -L common/stats # run tests in src/common/stats/test
$ ctest -L common # run tests in src/common/*/test

Run tests for N times until failed

$ ctest –repeat-until-fail N

Run failed test(s) of the last run

$ ctest –rerun-failed
